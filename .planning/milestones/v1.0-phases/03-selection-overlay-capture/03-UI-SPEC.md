# Phase 3 -- UI Design Contract

> Visual and interaction contract for the selection overlay, capture pipeline, and instruction input. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (vanilla HTML/CSS/TS in Electron overlay renderer) |
| Icon library | Inline SVG (hand-drawn, matching Phase 2 toolbar icons) |
| Font | System font stack: `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Helvetica, Arial, sans-serif` |

**Rationale:** The overlay is a transparent HTML page rendered in a WebContentsView on top of the user's website. Heavy component libraries and CSS frameworks are inappropriate here. All styling is vanilla CSS targeting a small surface area (toolbar, selection rectangle, hover highlight, instruction input bar). Phase 2 established this pattern and Phase 3 continues it.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, toolbar button gaps, inline padding |
| sm | 8px | Toolbar internal padding (horizontal), input field padding |
| md | 12px | Input bar padding, gap between input and submit button |
| lg | 16px | Toolbar margin from window edge, input bar margin from selection |
| xl | 24px | Not used in this phase |
| 2xl | 32px | Not used in this phase |

Exceptions:
- Toolbar button size: 36x36px (established in Phase 2, not changing)
- Toolbar pill padding: `10px 8px` (established in Phase 2, not changing)
- Input bar minimum offset from selection edge: 8px
- Input bar maximum width: 480px
- Input bar offset from viewport edge: 16px minimum (prevents clipping)

## Color

This phase's overlay is transparent by design -- it floats above the user's website. The traditional 60/30/10 model is adapted for an overlay context:

| Role | Value | Opacity | Usage |
|------|-------|---------|-------|
| Dominant (60%) | `transparent` | 0% | Overlay background. User's site shows through entirely. |
| UI Chrome (30%) | `rgb(10, 10, 10)` | 88% (`rgba(10, 10, 10, 0.88)`) | Toolbar background, instruction input bar background. Established in Phase 2. |
| Selection Accent (10%) | `rgb(138, 180, 248)` | See below | Selection rectangle border, element hover highlight border. Soft blue. |
| Destructive | Not applicable | -- | No destructive actions in this phase. |

### Selection accent detail

| Element | Border | Fill | Notes |
|---------|--------|------|-------|
| Freeform rectangle (drawing) | `rgba(138, 180, 248, 0.8)` 2px solid | `rgba(138, 180, 248, 0.08)` | Semi-transparent tint over selected content |
| Freeform rectangle (committed) | `rgba(138, 180, 248, 0.9)` 2px solid | `rgba(138, 180, 248, 0.10)` | Slightly more opaque once drawn |
| Element hover highlight | `rgba(138, 180, 248, 0.6)` 2px solid | `rgba(138, 180, 248, 0.06)` | Lighter than committed selection |
| Element click selection | `rgba(138, 180, 248, 0.9)` 2px solid | `rgba(138, 180, 248, 0.10)` | Matches committed rectangle selection |

**Border radius on all selection shapes:** `6px` (per D-05: rounded, not OS-sharp).

## Interaction States

### Mode: Inactive (default)

- Overlay bounds shrunk to toolbar area only (Phase 2 pattern: `setOverlayInactive`)
- Toolbar visible at bottom-right, opacity 0.75, opacity 1 on hover
- No selection rectangle visible, no crosshair cursor
- User interacts with their site normally (mouse events pass through)

### Mode: Rectangle Selection

**Entry:** User clicks the rectangle select button in toolbar.
**Cursor:** `crosshair` on the full overlay surface.
**Drawing:** User clicks and drags to draw a rectangle.

| State | Visual |
|-------|--------|
| Idle (mode active, no drag) | Crosshair cursor. Toolbar select button shows active state (accent icon + accent bg). |
| Drawing (mousedown + mousemove) | Selection rectangle grows from origin to current mouse position. Border + tint fill visible. |
| Committed (mouseup) | Selection rectangle stays visible. Instruction input bar appears. |
| Cancelled (Escape or toolbar button click) | Selection rectangle removed. Return to inactive mode. |

**Minimum selection size:** 16x16px. Selections smaller than this are ignored (treated as accidental click).

### Mode: Element Selection

**Entry:** User clicks the element select button in toolbar (second button, added in Phase 3).
**Cursor:** `default` (pointer). Element highlighting provides the visual feedback.

| State | Visual |
|-------|--------|
| Hovering | Element under cursor highlighted with hover highlight style (lighter accent). |
| Clicked | Highlight transitions to committed selection style (stronger accent). Instruction input bar appears. |
| Cancelled (Escape or toolbar button click) | Highlight removed. Return to inactive mode. |

**Hover detection:** Uses `document.elementFromPoint()` on the site view via `executeJavaScript`. Highlight rectangle positioned over the element's bounding rect.

### Instruction Input Bar

**Appearance:** Appears after selection is committed (rectangle drawn or element clicked).
**Position (per D-08):** Smart positioning near the selection.

| Condition | Position |
|-----------|----------|
| Space below selection >= 80px | Below selection, left-aligned with selection left edge |
| Space below < 80px, space above >= 80px | Above selection, left-aligned with selection left edge |
| Neither above nor below has 80px | Below selection, scrolled into view area if possible |

**Constraints:**
- Minimum 16px from any viewport edge
- Maximum width: 480px
- Minimum width: 240px
- Width matches selection width when selection is between 240px and 480px

**Structure:**

```
+-------------------------------------------------------+
|  [placeholder / typed text]                   [submit] |
+-------------------------------------------------------+
  Enter to submit, Shift+Enter for new line
```

| Property | Value |
|----------|-------|
| Background | `rgba(10, 10, 10, 0.88)` |
| Border | `1px solid rgba(255, 255, 255, 0.1)` |
| Border radius | `12px` |
| Padding | `12px 12px 12px 16px` |
| Min height | 40px (single line) |
| Max height | 160px (then scrolls internally) |
| Text area resize | None (auto-expanding) |
| Submit button size | 28x28px, inside the input bar, right-aligned |
| Shadow | `0 4px 24px rgba(0, 0, 0, 0.3)` |

**Keyboard hint:** Small text below the input bar, `12px rgba(255, 255, 255, 0.35)`, reads: `Enter to submit . Shift+Enter for new line`

### Toolbar (extended from Phase 2)

Phase 3 adds a second button to the existing toolbar pill:

```
+---------+
| [drag]  |   <- existing drag handle
+---------+
| [rect]  |   <- existing select button, becomes rectangle mode
+---------+
| [elem]  |   <- NEW: element select mode button
+---------+
```

| Property | Value |
|----------|-------|
| Element button icon | Inline SVG: pointer cursor with element bounding box (similar to browser inspector icon) |
| Active state | When either mode is active, that button's icon color changes to accent, button bg changes to accent at 15% opacity |
| Mutually exclusive | Only one mode active at a time. Clicking the other mode switches. Clicking the active mode deactivates. |

## Animation & Transition Contract

| Element | Property | Duration | Easing |
|---------|----------|----------|--------|
| Toolbar opacity | opacity | 150ms | ease |
| Toolbar button hover | color, background | 120ms | ease |
| Selection rectangle appearance | none (immediate draw during drag) | 0ms | -- |
| Element hover highlight | opacity | 80ms | ease-out |
| Input bar appearance | opacity, transform (translateY 4px -> 0) | 150ms | ease-out |
| Input bar disappearance | opacity | 100ms | ease-in |
| Input bar auto-expand | height | 100ms | ease-out |

**No spring animations. No bouncing. No delays.** The overlay must feel instant and non-intrusive. Transitions exist only to avoid jarring flashes, not for delight.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | Escape cancels selection in any mode. Tab focuses submit button inside input bar. |
| Screen reader labels | All toolbar buttons have `aria-label`. Input bar textarea has `aria-label="Change instruction"`. |
| Focus management | When input bar appears, textarea receives focus automatically. When input bar dismisses, focus returns to toolbar. |
| Color contrast | White text on dark chrome: contrast ratio >= 7:1 (WCAG AAA). Accent blue on dark chrome: contrast ratio >= 4.5:1 (WCAG AA). |
| Motion sensitivity | All transitions under 200ms. No looping animations. `prefers-reduced-motion: reduce` disables all transitions. |

## DPI / Retina Contract (CAP-03)

| Concern | Implementation |
|---------|----------------|
| Selection coordinates | All selection coordinates in the overlay are CSS pixels (logical pixels). |
| Screenshot capture | Main process reads `siteView.webContents.getZoomFactor()` and `screen.getPrimaryDisplay().scaleFactor` to compute device pixel coordinates before calling `capturePage(rect)`. |
| NativeImage crop | `capturePage()` returns a NativeImage at device pixel resolution. Crop coordinates must be multiplied by `devicePixelRatio`. |
| Output format | PNG buffer via `NativeImage.toPNG()`. No JPEG, no lossy compression. Claude needs pixel-accurate context. |

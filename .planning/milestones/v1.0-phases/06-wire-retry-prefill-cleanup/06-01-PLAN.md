<objective>
Wire the retry-prefill flow so that clicking "Retry" on a failed sidebar task prefills the overlay textarea with the original instruction and activates selection mode, instead of silently auto-resubmitting.

Purpose: Closes the `retry-prefill-flow` gap identified in the v1.0 milestone audit. The current retry handler calls `agentManager.retryTask()` which auto-resubmits with the original context. The user never gets a chance to edit the instruction or make a new selection. This plan gives the user control over both.

Output: Modified `ipc-handlers.ts` (main process) and `overlay.ts` (renderer) that together implement the full retry-prefill-resubmit UX path.
</objective>

**Replacement code:**
```typescript
// Sidebar task retry -- prefill overlay with original instruction, activate selection mode
// User can edit instruction and make a new selection before re-submitting (gap closure: retry-prefill-flow)
ipcMain.handle('sidebar:task-retry', async (_event, data: { id: string }) => {
  const task = agentManager.getTask(data.id);
  if (!task) return;

  const instruction = task.instruction;

  // Dismiss the old error task (instruction is preserved in prefilled textarea)
  agentManager.dismissTask(data.id);
  // If no tasks remain after dismissal, hide sidebar
  if (agentManager.getAllTasks().length === 0) {
    components.setSidebarState('hidden');
  }

  // Prefill the overlay textarea with the original instruction
  components.overlayView.webContents.send('overlay:prefill-instruction', {
    instruction,
  });

  // Activate overlay for new selection (expands bounds to full window, minimizes sidebar)
  setOverlayActive(components.overlayView, components.window, components);

  // Enter rect-idle selection mode (default, more general than elem-idle)
  components.overlayView.webContents.send('overlay:mode-change', 'selection');
});
```

**Key behaviors:**
1. Reads the original task instruction via `agentManager.getTask(data.id)` before dismissing
2. Returns early (no-op) if task not found (handles race condition of rapid clicks)
3. Dismisses the error task from sidebar AFTER reading instruction
4. Hides sidebar if no tasks remain (consistent with `sidebar:task-dismiss` handler pattern at lines 152-158)
5. Sends `overlay:prefill-instruction` IPC with `{ instruction }` payload
6. Calls `setOverlayActive()` to expand overlay bounds to full window (same function used by `overlay:activate-selection` handler)
7. Sends `overlay:mode-change` with `'selection'` to notify renderer

**Important:** Do NOT call `agentManager.retryTask()` -- that auto-resubmits. The whole point is to let the user edit and re-submit manually.

**Replacement code:**
```typescript
// Wire prefill for retry: when retry is clicked in sidebar, the overlay receives the
// original instruction to pre-populate the textarea AND activates selection mode so
// the user can make a new selection and edit the instruction before re-submitting.
if (window.claw?.onPrefillInstruction) {
  window.claw.onPrefillInstruction((data: { instruction: string }) => {
    const ta = document.getElementById('claw-input-textarea') as HTMLTextAreaElement;
    if (ta) {
      ta.value = data.instruction;
      // Trigger auto-expand and enable submit button
      ta.dispatchEvent(new Event('input'));
    }

    // Activate overlay surface for interaction (body class + toolbar pinning)
    activateOverlaySurface();

    // Enter rect-idle selection mode so user can draw a new selection
    dispatch({ type: 'ACTIVATE_RECT' });

    // Show input bar at default position (no selection yet -- center at 60% height)
    const inputBar = document.getElementById('claw-input-bar')!;
    inputBar.hidden = false;
    inputBar.style.left = '50%';
    inputBar.style.transform = 'translateX(-50%)';
    inputBar.style.top = `${Math.round(window.innerHeight * 0.6)}px`;
    inputBar.style.bottom = '';
    requestAnimationFrame(() => {
      inputBar.classList.add('claw-input-bar--visible');
    });

    // Focus textarea so user can immediately edit (accessibility: keyboard focus)
    if (ta) {
      ta.focus();
    }
  });
}
```

**Key behaviors:**
1. Sets textarea value and dispatches `input` event (triggers auto-expand + enables submit button) -- SAME as before
2. Calls `activateOverlaySurface()` -- adds `body.claw-overlay--active` class, pins toolbar at current position
3. Dispatches `{ type: 'ACTIVATE_RECT' }` to state machine -- enters `rect-idle` mode (user can draw a selection)
4. Manually shows the input bar at a default centered position: `left: 50%`, `transform: translateX(-50%)`, `top: 60%` of viewport height. This does NOT use `showInputBar(bounds)` because there is no selection bounds yet. Once the user commits a selection, the existing `claw:selection-committed` listener (line 658) will reposition the input bar relative to the selection.
5. Uses `requestAnimationFrame` before adding `claw-input-bar--visible` class for the entrance animation (consistent with existing pattern in `showInputBar`)
6. Focuses the textarea for keyboard accessibility

**Why not use `showInputBar()`:** The existing `showInputBar(bounds)` function requires a `SelectionBounds` argument and resets `textarea.value = ''` (line 513). We need to preserve the prefilled text and position without bounds, so we inline the positioning logic.

**Important edge case (Pitfall 2):** After prefill, the state machine is in `rect-idle` but `state.selectionBounds` is null. The `handleSubmit()` function (line 609) guards with `if (!state.selectionBounds) return;`, so the user MUST draw a selection before submitting. This is the correct behavior -- submit is disabled until selection is made even though the button appears enabled (it has text content). The user sees an enabled-looking submit button but clicking it without a selection is a no-op. This matches the normal flow.

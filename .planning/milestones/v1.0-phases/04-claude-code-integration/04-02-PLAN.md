    Create `src/renderer/sidebar.ts` -- the main sidebar renderer logic.

    Declare the sidebar API type for TypeScript (since contextBridge creates it at runtime):

    ```typescript
    declare global {
      interface Window {
        clawSidebar: {
          onTaskUpdate: (cb: (data: TaskUpdate) => void) => void;
          expand: () => Promise<void>;
          collapse: () => Promise<void>;
          dismissTask: (id: string) => Promise<void>;
          retryTask: (id: string) => Promise<void>;
        };
      }
    }
    ```

    **Task data model:**
    ```typescript
    interface TaskUpdate {
      id: string;
      instruction: string;
      status: 'queued' | 'sending' | 'editing' | 'done' | 'error';
      error?: string;
    }
    ```

    Keep a `Map<string, TaskUpdate>` as local state for all tasks.

    **Core functions:**

    1. `renderTask(task: TaskUpdate)`: Creates or updates a task row DOM element.
       - Task row: div.task-row with `data-task-id` attribute
       - Instruction text: div.task-instruction (truncated to 2 lines via CSS)
       - Status row: flex container with status badge + dismiss button
       - Status badge: span.status-badge with class matching status (queued/sending/editing/done/error)
       - Badge text from copywriting contract: "Queued", "Sending", "Editing", "Done", "Error"
       - Dismiss button (done/error only): "x" icon button, 20x20px, class `task-row-dismiss-btn`, visible on hover
       - For error rows: add `.error-row` class to instruction text, append error message div.task-error-message, and button row with Retry + Dismiss text buttons
       - If task row already exists in DOM (update), update badge class and text, add/remove error elements as needed
       - If new task row, prepend to task list (newest on top)

    2. `updateBadge()`: Count completed (done + error) and total tasks, update `#sidebar-badge-text` to `"{completed}/{total}"` format. Show badge if total > 0, hide if total === 0.

    3. `triggerBadgePulse(type: 'done' | 'error')`: Add `pulse-done` or `pulse-error` class to badge. Remove after 600ms via animationend listener.

    4. `handleExpand()`: Call `window.clawSidebar.expand()`. Show `#sidebar-expanded`, hide `#sidebar-minimized`.

    5. `handleCollapse()`: Call `window.clawSidebar.collapse()`. Hide `#sidebar-expanded`, show `#sidebar-minimized`.

    6. `autoExpandBrief()`: Show expanded, set 2000ms timer, then collapse. If user interacts (mouseenter on expanded panel), clear the timer. Per D-06.

    **Event wiring (on DOMContentLoaded):**

    1. `window.clawSidebar.onTaskUpdate((data) => { ... })`:
       - Add/update task in local Map
       - Call `renderTask(data)`
       - Call `updateBadge()`
       - If this is a new task (not in map before): call `autoExpandBrief()` per D-06
       - If task completed/errored and sidebar is minimized: call `triggerBadgePulse(data.status === 'error' ? 'error' : 'done')` per D-07

    2. `#sidebar-expand-btn` click -> `handleExpand()`
    3. `#sidebar-minimize-btn` click -> `handleCollapse()`
    4. Task row retry click -> `window.clawSidebar.retryTask(taskId)`
    5. Task row dismiss click -> `window.clawSidebar.dismissTask(taskId)`, remove row from DOM, remove from local Map, call `updateBadge()`. If no tasks remain, the main process will set bounds to hidden.

    **Sidebar state tracking:**
    Track `sidebarState: 'hidden' | 'minimized' | 'expanded'` locally. The main process manages actual bounds, but the renderer needs to know its visual state for conditional logic (e.g., badge pulse only when minimized).

    **Accessibility:**
    - Tab order: minimize button, then task rows (retry/dismiss buttons within each row)
    - `aria-live="polite"` on task list container (already in HTML)
    - Each task row: `aria-label="{instruction text}, status: {status}"` updated on render
  </action>

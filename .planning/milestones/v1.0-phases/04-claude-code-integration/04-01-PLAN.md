    Create `src/main/agent-manager.ts`:

    ```typescript
    export type TaskStatus = 'queued' | 'sending' | 'editing' | 'done' | 'error';

    export interface Task {
      id: string;
      instruction: string;
      status: TaskStatus;
      error?: string;
      screenshot: Buffer;
      dom: DomExtractionResult;
      bounds: CSSRect;
    }

    export interface TaskUpdate {
      id: string;
      instruction: string;
      status: TaskStatus;
      error?: string;
    }

    export class AgentManager {
      private tasks: Map<string, InternalTask> = new Map();
      private activeCount = 0;
      private readonly maxParallel = 3;
      private readonly projectDir: string;
      private onTaskUpdate: ((update: TaskUpdate) => void) | null = null;

      constructor(projectDir: string);
      setOnTaskUpdate(cb: (update: TaskUpdate) => void): void;
      async submitTask(input: { instruction: string; screenshot: Buffer; dom: DomExtractionResult; bounds: CSSRect }): Promise<string>;
      retryTask(id: string, freshScreenshot: Buffer, freshDom: DomExtractionResult, freshBounds: CSSRect): Promise<string>;
      dismissTask(id: string): void;
      shutdown(): void;
      getTask(id: string): Task | undefined;
      getAllTasks(): Task[];
    }
    ```

    InternalTask extends Task with private fields: `abortController?: AbortController`, `queryRef?: { close: () => void }`.

    Key implementation details:
    1. `submitTask`: generates ID via `crypto.randomUUID()`, creates task with status 'queued', calls `emitUpdate()`, calls `processQueue()`
    2. `processQueue`: if `activeCount < maxParallel`, finds first 'queued' task, increments activeCount, sets 'sending', calls `executeTask()`. On completion/error, decrements activeCount, calls `processQueue()` again for queue drain.
    3. `executeTask`: creates AbortController, calls `query()` from `@anthropic-ai/claude-agent-sdk` with:
       - `prompt: assemblePrompt(task.instruction, task.screenshot, task.dom)`
       - `options.abortController: abortController`
       - `options.cwd: this.projectDir`
       - `options.systemPrompt: { type: 'preset', preset: 'claude_code', append: 'You are used by claw-design. The user selected a region of their website and provided a screenshot, DOM context, and change instruction. Edit the source code to implement the change. Be concise in your response.' }`
       - `options.allowedTools: ['Read', 'Write', 'Edit', 'Glob', 'Grep', 'Bash']`
       - `options.permissionMode: 'acceptEdits'`
       - `options.settingSources: ['project']`
       - `options.persistSession: false`
       - `options.maxTurns: 20`
    4. Status mapping from SDK messages (in for-await loop):
       - `message.type === 'system' && message.subtype === 'init'` -> task status = 'editing'
       - `message.type === 'result' && message.subtype === 'success'` -> task status = 'done'
       - `message.type === 'result' && message.subtype !== 'success'` -> task status = 'error'
       - All other message types are ignored (prevents IPC flooding per Pitfall 3)
    5. Error message extraction: map SDK error strings to user-friendly messages:
       - `'authentication_failed'` -> `'Authentication failed. Check your Claude API key.'`
       - `'rate_limit'` -> `'Rate limit reached. Retry in a moment.'`
       - `'billing_error'` -> `'Billing issue. Check your Claude account.'`
       - `'server_error'` -> `'Claude server error. Retry in a moment.'`
       - default -> `'Something went wrong. Retry or dismiss to continue.'`
    6. Wrap for-await in try/catch/finally. Finally: `q.close()` and decrement activeCount.
    7. `retryTask`: reads original instruction from task, creates new task via submitTask with original instruction but fresh screenshot/dom/bounds.
    8. `dismissTask`: if task is active (sending/editing), call `abortController.abort()`. Delete from map. Emit update is not needed since task is removed.
    9. `shutdown`: iterate all tasks, abort active ones, close all query refs.

    Create `tests/main/agent-manager.test.ts`:
    - Mock `@anthropic-ai/claude-agent-sdk` using `vi.mock()`. The mock `query()` returns an object that is both an async iterable (yielding configurable SDK messages) and has a `.close()` method.
    - Create a helper `createMockQuery(messages: SDKMessage[])` that returns `{ [Symbol.asyncIterator]: ..., close: vi.fn() }`.
    - Use `vi.fn()` for onTaskUpdate callback to capture all status emissions.
    - Test concurrency by submitting 4 tasks with slow-resolving mock queries (use Promises that resolve on demand).

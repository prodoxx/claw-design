**Create `src/main/window.ts`** -- BaseWindow factory with dual WebContentsView (per D-09):

The module exports three functions: `createMainWindow`, `setOverlayInactive`, `setOverlayActive`.

`createMainWindow(url: string, projectName: string, port: number)` does:
1. Create `BaseWindow` with `width: 1280, height: 800, center: true` (per D-02). Standard frame (per D-01 -- `frame` defaults to true, do NOT set it explicitly). Set `title` to `claw-design \u2014 ${projectName} \u2014 localhost:${port}` (per D-03). No minimum size constraint (per D-04).
2. Create site `WebContentsView` with secure webPreferences: `contextIsolation: true, sandbox: true, nodeIntegration: false, webSecurity: true, allowRunningInsecureContent: false`. No preload for site view.
3. Add site view as child of `win.contentView` via `addChildView(siteView)`.
4. Load `url` (the full http://localhost:PORT string) in site view via `siteView.webContents.loadURL(url)`.
5. Create overlay `WebContentsView` with secure webPreferences: `contextIsolation: true, sandbox: true, nodeIntegration: false, webSecurity: true, preload: path.join(__dirname, '../preload/overlay.mjs')`.
6. Call `overlayView.webContents.setBackgroundColor('#00000000')` IMMEDIATELY after creation and BEFORE loading content (per Pitfall 4 from research).
7. Add overlay view as child via `addChildView(overlayView)`.
8. Load overlay HTML: `overlayView.webContents.loadFile(path.join(__dirname, '../renderer/overlay.html'))`.
9. Create a `syncBounds()` function that reads `win.getContentBounds()` and sets both views' bounds to `{ x: 0, y: 0, width, height }` for the site view, and calls `setOverlayInactive` for the overlay view (so it only covers the indicator area).
10. Listen to `win.on('resize', syncBounds)` (per D-13, Pitfall 3).
11. Call `syncBounds()` immediately.
12. Return an object: `{ window: win, siteView, overlayView }`.

`setOverlayInactive(overlayView: WebContentsView, win: BaseWindow)` -- Shrinks overlay bounds to only cover the bottom-right indicator area (48x48 px). Per D-12: when overlay is inactive, mouse events pass through to site view. This works because the overlay's bounds no longer cover the site.

```typescript
export function setOverlayInactive(overlayView: WebContentsView, win: BaseWindow): void {
  const { width, height } = win.getContentBounds();
  const indicatorSize = 48;
  overlayView.setBounds({
    x: width - indicatorSize,
    y: height - indicatorSize,
    width: indicatorSize,
    height: indicatorSize,
  });
}
```

`setOverlayActive(overlayView: WebContentsView, win: BaseWindow)` -- Expands overlay to full window size for selection mode (Phase 3 triggers this).

```typescript
export function setOverlayActive(overlayView: WebContentsView, win: BaseWindow): void {
  const { width, height } = win.getContentBounds();
  overlayView.setBounds({ x: 0, y: 0, width, height });
}
```

Also export a `WindowComponents` type:
```typescript
export interface WindowComponents {
  window: BaseWindow;
  siteView: WebContentsView;
  overlayView: WebContentsView;
}
```

**Create `src/main/navigation.ts`** -- Navigation restriction (per D-05, D-06):

Export `setupNavigation(siteView: WebContentsView, allowedOrigin: string): void`.

1. Listen to `siteView.webContents.on('will-navigate', (event, navigationUrl) => { ... })`. Parse `navigationUrl` with `new URL()`. If `parsed.origin !== allowedOrigin`, call `event.preventDefault()` and `shell.openExternal(navigationUrl)`. Otherwise allow (per D-05).
2. Call `siteView.webContents.setWindowOpenHandler(({ url }) => { ... })`. Parse url. If `parsed.origin !== allowedOrigin`, call `shell.openExternal(url)`. Always return `{ action: 'deny' }` -- never open new Electron windows, even for localhost (per D-06).
3. Wrap URL parsing in try/catch -- if URL is malformed, prevent navigation and log a warning.

Import `shell` from 'electron'.

**Create `src/main/ipc-handlers.ts`** -- IPC handler registrations:

Export `registerIpcHandlers(components: WindowComponents): void`.

2. Also register a `overlay:deactivate-selection` handler that calls `setOverlayInactive` and sends `'inactive'` mode. Phase 3 will call this when selection is complete.

Import `ipcMain` from 'electron' and `setOverlayActive, setOverlayInactive, type WindowComponents` from './window.js'.

**Create `src/main/index.ts`** -- Electron main process entry point:

```typescript
import { app } from 'electron';
import { createMainWindow, setOverlayInactive } from './window.js';
import { setupNavigation } from './navigation.js';
import { registerIpcHandlers } from './ipc-handlers.js';

const url = process.env.CLAW_URL ?? 'http://localhost:3000';
const projectName = process.env.CLAW_PROJECT_NAME ?? 'unknown';
const port = parseInt(url.match(/:(\d+)/)?.[1] ?? '3000', 10);

app.whenReady().then(() => {
  const components = createMainWindow(url, projectName, port);

  // Setup navigation restriction (D-05, D-06)
  const allowedOrigin = new URL(url).origin;
  setupNavigation(components.siteView, allowedOrigin);

  // Register IPC handlers for overlay communication
  registerIpcHandlers(components);

  // Start with overlay in inactive state (D-12)
  setOverlayInactive(components.overlayView, components.window);

  // Handle window close
  components.window.on('closed', () => {
    app.quit();
  });
});

// Quit when all windows are closed (macOS included)
app.on('window-all-closed', () => {
  app.quit();
});
```

Note: Do NOT call `app.dock.hide()` or modify dock behavior. Standard app behavior.

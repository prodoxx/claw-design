    **1. Create pure state machine module in `src/renderer/overlay.ts`:**

    Define the state machine types and pure transition function at the top of the file (exported for testing):

    ```typescript
    export type OverlayMode = 'inactive' | 'rect-idle' | 'rect-drawing' | 'rect-committed'
                            | 'elem-idle' | 'elem-hovering' | 'elem-committed';

    export interface SelectionBounds {
      x: number; y: number; width: number; height: number;
    }

    export interface SelectionState {
      mode: OverlayMode;
      startX: number;
      startY: number;
      currentX: number;
      currentY: number;
      hoveredRect: SelectionBounds | null;
      selectionBounds: SelectionBounds | null;
    }

    export type SelectionEvent =
      | { type: 'ACTIVATE_RECT' }
      | { type: 'ACTIVATE_ELEM' }
      | { type: 'MOUSE_DOWN'; x: number; y: number }
      | { type: 'MOUSE_MOVE'; x: number; y: number }
      | { type: 'MOUSE_UP'; x: number; y: number }
      | { type: 'ELEMENT_HOVER'; rect: SelectionBounds }
      | { type: 'ELEMENT_CLICK' }
      | { type: 'CANCEL' };

    export const INITIAL_STATE: SelectionState = {
      mode: 'inactive', startX: 0, startY: 0, currentX: 0, currentY: 0,
      hoveredRect: null, selectionBounds: null,
    };

    export const MIN_SELECTION_SIZE = 16; // per UI spec

    export function transition(state: SelectionState, event: SelectionEvent): SelectionState { ... }
    ```

    Transition logic:
    - `inactive` + `ACTIVATE_RECT` -> `rect-idle`
    - `inactive` + `ACTIVATE_ELEM` -> `elem-idle`
    - `rect-idle` + `MOUSE_DOWN` -> `rect-drawing` (set startX, startY, currentX, currentY from event)
    - `rect-drawing` + `MOUSE_MOVE` -> `rect-drawing` (update currentX, currentY)
    - `rect-drawing` + `MOUSE_UP` -> if width >= 16 && height >= 16: `rect-committed` with selectionBounds computed from min/max of start/end. Else: `rect-idle` (too small).
    - `elem-idle` + `ELEMENT_HOVER` -> `elem-hovering` with hoveredRect from event
    - `elem-hovering` + `ELEMENT_HOVER` -> `elem-hovering` with updated hoveredRect
    - `elem-hovering` + `ELEMENT_CLICK` -> `elem-committed` with selectionBounds = hoveredRect
    - Any active mode + `CANCEL` -> `inactive` with all selection data cleared
    - `rect-committed` or `elem-committed` + `ACTIVATE_RECT` -> `rect-idle` (switch mode, clear selection)
    - `rect-committed` or `elem-committed` + `ACTIVATE_ELEM` -> `elem-idle` (switch mode, clear selection)
    - Ignore invalid transitions (return state unchanged)

    **2. Wire DOM event listeners in `src/renderer/overlay.ts`:**

    Replace the Phase 2 scaffold with full selection logic:
    - `mousedown` on overlay document: if mode is `rect-idle`, dispatch `MOUSE_DOWN`
    - `mousemove` on overlay document: if mode is `rect-drawing`, dispatch `MOUSE_MOVE`. If mode is `elem-idle` or `elem-hovering`, throttle via requestAnimationFrame and call `window.claw.getElementAtPoint(e.clientX, e.clientY)`, then dispatch `ELEMENT_HOVER` with the returned rect.
    - `mouseup` on overlay document: if mode is `rect-drawing`, dispatch `MOUSE_UP`
    - `click` on overlay document: if mode is `elem-hovering`, dispatch `ELEMENT_CLICK`
    - `keydown` Escape: dispatch `CANCEL`, then call `window.claw.deactivateSelection()` per D-03
    - Rectangle select button click: if inactive, dispatch `ACTIVATE_RECT` + call `window.claw.activateSelection()`. If already rect mode, dispatch `CANCEL` + call `window.claw.deactivateSelection()`. Per D-03 (toggle behavior).
    - Element select button click: if inactive, dispatch `ACTIVATE_ELEM` + call `window.claw.activateSelection()`. If already elem mode, dispatch `CANCEL` + call `window.claw.deactivateSelection()`. Per D-02/D-03.
    - If switching from one active mode to the other, call `ACTIVATE_RECT` or `ACTIVATE_ELEM` without deactivating (overlay stays full-window).

    **3. Render function in `src/renderer/overlay.ts`:**

    After each state transition, call `render(state)` which:
    - Sets cursor on `document.body`: `crosshair` for rect-idle/rect-drawing, `default` otherwise (per UI spec cursor contract)
    - Shows/hides `#claw-selection-rect` div and positions it based on startX/startY/currentX/currentY (for rect modes) or selectionBounds (for committed)
    - Shows/hides `#claw-element-highlight` div and positions it based on hoveredRect (for elem modes) or selectionBounds (for elem-committed)
    - Updates toolbar button active states: `.claw-toolbar-btn--active` class on the active mode button
    - Updates selection rect CSS classes: `.claw-selection-rect--drawing` during drag, `.claw-selection-rect--committed` after mouseup
    - When mode is `rect-committed` or `elem-committed`, dispatches a custom event `claw:selection-committed` with `detail: { bounds: state.selectionBounds }` on the document. Plan 03's input bar listens for this.

    **6. Add IPC handler for element detection in `src/main/ipc-handlers.ts`:**

    Add inside `registerIpcHandlers()`:
    ```typescript
    ipcMain.handle('overlay:get-element-at-point', async (_event, x: number, y: number) => {
      const result = await components.siteView.webContents.executeJavaScript(`
        (function() {
          const el = document.elementFromPoint(${x}, ${y});
          if (!el) return null;
          const rect = el.getBoundingClientRect();
          return {
            x: Math.round(rect.x),
            y: Math.round(rect.y),
            width: Math.round(rect.width),
            height: Math.round(rect.height)
          };
        })()
      `);
      return result;
    });
    ```

    Per D-04: top-level document only, no shadow DOM or iframe traversal.

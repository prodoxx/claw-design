    Create the CLI entry point, start command skeleton, dev server utilities, and a terminal output helper module.

    **src/cli/utils/output.ts** (terminal output helpers per D-01):
    - Export `createSpinner(text: string)` that wraps `ora(text).start()` -- returns the ora spinner instance
    - Export `printHeader(version: string)` that prints `clawdesign v{version}` using `picocolors.bold`
    - Export `printReady(port: number)` that prints the ready message:
      ```
        Ready!
        Dev server: http://localhost:{port}
        Press Ctrl+C to stop
      ```
      Use `picocolors.green` for "Ready!", `picocolors.dim` for the rest
    - Export `printError(title: string, message: string, suggestion?: string)` that prints structured error with red cross mark, description, and dim suggestion (per D-06 error style)

    **src/cli/utils/dev-server.ts:**
    - Export `const SCRIPT_PRIORITY = ['dev', 'start', 'serve'] as const`
    - Export class `DetectionError extends Error` with name set to `'DetectionError'`
    - Export interface `DetectedScript { name: string; command: string }`
    - Export `async function detectDevServerScript(projectDir: string): Promise<DetectedScript>`:
      - Read `{projectDir}/package.json` using `readFile` from `node:fs/promises`
      - Parse JSON, check for `scripts` field
      - If no `scripts` field: throw `DetectionError('No "scripts" field found in package.json')`
      - Iterate `SCRIPT_PRIORITY`, return first match as `{ name, command: pkg.scripts[name] }`
      - If no match: throw `DetectionError` with message listing what was looked for and suggesting `--cmd` flag (per D-06):
        ```
        No dev server script found in package.json.
          Looked for: dev, start, serve
          Tip: use --cmd "your-dev-command" to specify manually
        ```
    - Export `function spawnDevServer(command: string, verbose: boolean): ChildProcess`:
      - Use `spawn(command, { shell: true, stdio: ['ignore', 'pipe', 'pipe'], cwd: process.cwd() })` (per research Pattern 3, shell: true per Pitfall 1)
      - If `verbose`, pipe child.stdout to process.stdout and child.stderr to process.stderr
      - Return the child process

    **src/cli/commands/start.ts:**
    - Export interface `StartOptions { cmd?: string; port?: string; verbose?: boolean }`
    - Export `async function startCommand(options: StartOptions): Promise<void>` as a skeleton:
      - For now, implement Steps 1-2 only (detect and spawn dev server)
      - Step 1: If `options.cmd` provided, use it directly. Else call `detectDevServerScript(process.cwd())`
      - Step 2: Call `spawnDevServer(command, options.verbose ?? false)`
      - Leave a `// TODO: Port detection, Claude session, Electron launch (Plan 02 & 03)` comment
      - Log a temporary "Dev server spawned" message
      - Register a temporary SIGINT handler that kills the dev server child with tree-kill

    **src/cli/index.ts** (per D-03, D-04):
    - Shebang: `#!/usr/bin/env node`
    - Import `Command` from `commander`
    - Import `startCommand` from `./commands/start.js`

    - Create program: `new Command()`, name `'clawdesign'` (D-03), description `'Visual web development tool powered by Claude Code'`, version from package.json
    - Add `start` subcommand with options: `--cmd <command>` (D-03), `--port <number>` (D-08), `--verbose` (D-02)
    - Call `program.parse()`
    - Per D-04: only `start`, `--version`, `--help` supported

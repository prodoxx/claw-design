    **Imports:**
    ```typescript
    import { detectDevServerScript, spawnDevServer, DetectionError } from '../utils/dev-server.js';
    import { extractPortFromOutput, waitForPort, getProcessOnPort } from '../utils/port-detect.js';

    import pc from 'picocolors';
    import type { ChildProcess } from 'node:child_process';
    ```

    **Interface:** Keep `StartOptions { cmd?: string; port?: string; verbose?: boolean }`.

    **startCommand(options) implementation -- 7 steps:**

    **Step 1: Check Claude Code installed (per D-12)**
    - Call `isClaudeInstalled()`
    - If false: call `printError('Claude Code not found', 'Claude Code CLI is not installed or not in PATH.', 'Install: https://claude.ai/download')` and `process.exit(1)`

    **Step 2: Detect dev server (per D-05, D-06)**
    - Create spinner: `createSpinner('Detecting dev server...')`
    - If `options.cmd` provided: use it directly, spinner.succeed with `Using custom command: {cmd}` (cmd in cyan via `pc.cyan`)
    - Else: try `detectDevServerScript(process.cwd())`
      - On success: set command to `npm run ${detected.name}`, spinner.succeed with `Detected: npm run {name} (from package.json)` (command in cyan)
      - On DetectionError: spinner.fail('No dev server detected'), `printError('No dev server detected', err.message)`, `process.exit(1)`

    **Step 3: Spawn dev server (per CLI-04)**
    - Create spinner: `createSpinner('Starting dev server...')`
    - Call `spawnDevServer(command, options.verbose ?? false)`
    - spinner.succeed with `Dev server started` and the command in dim

    **Step 4: Detect port (per D-07, D-08)**
    - If `options.port` provided: parse it as integer, use directly
    - Else: listen to devServer.stdout and devServer.stderr data events, accumulate output, call `extractPortFromOutput(accumulatedOutput)` on each chunk until a port is found
      - Use a Promise that resolves when port is found or rejects after 10 seconds of no port detection
      - If no port detected from stdout: use `readline` from `node:readline` to prompt user for port interactively (per D-07: "prompt user interactively to specify the port")
    - If port is detected from stdout, update spinner text to show detected port

    **Step 5: Wait for port readiness (per D-09, D-14, D-15)**
    - Update spinner: `Waiting for localhost:${port}...`
    - Show elapsed time in spinner (update spinner.text every second with elapsed seconds)
    - Try `waitForPort(port, { timeout: 30_000 })`
    - On success: spinner.succeed with `Dev server ready on http://localhost:${port}` (URL in cyan)
    - On timeout error:
      - Check if port is in use by another process: `getProcessOnPort(port)`
      - If process found (per D-14): `printError('Port already in use', 'Port ${port} is occupied by ${name} (PID ${pid})', 'Try: kill ${pid} or clawdesign start --port <other>')`
      - If no process found (per D-15): `printError('Startup timeout', 'Port ${port} not ready after 30s', 'Try: clawdesign start --verbose\n  Or:  clawdesign start --port <port>')`
      - `process.exit(1)`

    **Step 6: Launch Claude Code session (per D-10: eager, per D-12: already checked)**
    - Create spinner: `createSpinner('Launching Claude Code...')`
    - Try `await spawnClaudeSession(process.cwd())`
    - On success: spinner.succeed with `Claude Code session ready`
    - On error: spinner.fail, `printError('Claude Code failed', err.message)`, `process.exit(1)`

    **Step 7: Register shutdown and print ready (per D-01 final step)**
    - Call `registerShutdownHandlers({ devServer: { pid: devServer.pid! }, claudeSession: claude })`
    - Listen for devServer `'exit'` event: when fired, log `pc.yellow('\n  Dev server exited.')` and `pc.dim('  Press Ctrl+C to stop, then re-run clawdesign start')` (per D-13: notify, keep running)
    - Call `printReady(port)`

    **Dev server crash handling (per D-13):**
    - Attach `devServer.on('exit', (code, signal) => { ... })` handler that prints warning but does NOT trigger shutdown. The user manually presses Ctrl+C to exit.

    The function must NOT call `process.exit(0)` at the end -- it should return and let the event loop keep the process alive (dev server and Claude session are long-running).

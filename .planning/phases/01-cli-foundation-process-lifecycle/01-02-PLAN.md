    **src/cli/utils/port-detect.ts:**

    Import `createConnection` from `node:net` and `execFileSync` from `node:child_process`.

    Export `PORT_PATTERNS` as a const array of RegExp objects, ordered from most specific to least specific (per Pitfall 4 in research -- order matters to avoid false positives):
    ```typescript
    export const PORT_PATTERNS: readonly RegExp[] = [
      // Full URL: http://localhost:3000, https://127.0.0.1:8080, http://0.0.0.0:4321, http://[::]:3000
      /https?:\/\/(?:localhost|127\.0\.0\.1|0\.0\.0\.0|\[::\]):(\d+)/,
      // "listening on port 3000", "running at port 8080", "started server on 0.0.0.0:3000"
      /(?:listening|running|started\s+server)\s+(?:on|at)\s+(?:(?:\S+:)?(?:port\s+)?)?(\d+)/i,
      // "port: 3000", "Port 3000", "PORT=3000"
      /\b(?:port|Port|PORT)\s*[:=]?\s*(\d+)/,
      // Fallback: any :NNNNN (4-5 digit port after colon) -- most generic, last resort
      /:(\d{4,5})\b/,
    ] as const;
    ```

    Export `function extractPortFromOutput(output: string): number | null`:
    - Iterate PORT_PATTERNS in order
    - For each pattern, test against the full output string
    - On first match, parse the captured group as integer
    - Validate port is > 0 and <= 65535
    - Return the port number, or null if no pattern matches

    Export `function waitForPort(port: number, opts?: { timeout?: number; interval?: number }): Promise<void>`:
    - Default timeout: 30000 (30s per D-09)
    - Default interval: 250ms
    - Create a polling loop using `net.createConnection({ port, host: '127.0.0.1' })`
    - On `'connect'` event: destroy socket, resolve
    - On `'error'` event: destroy socket, setTimeout for next attempt
    - If elapsed time exceeds timeout: reject with `Error(`Timeout: port ${port} not ready after ${timeout / 1000}s`)`

    Export `function getProcessOnPort(port: number): { pid: number; name: string } | null` (per D-14: port already in use):
    - Use `execFileSync('lsof', ['-i', `:${port}`, '-t'], { encoding: 'utf-8' })` to get PID
    - Use `execFileSync('ps', ['-p', String(pid), '-o', 'comm='], { encoding: 'utf-8' })` to get process name
    - Wrap in try/catch, return null on any error (lsof not available, no process, etc.)

    Export interface `ClaudeSession`:

    export interface ClaudeSession {

    Export `async function spawnClaudeSession(cwd: string): Promise<ClaudeSession>` (per D-10: eager spawn):
    - Create an async generator `messageStream` using a queue pattern:
      - Maintain `messageQueue: any[]` and `resolveNext: ((msg: any) => void) | null`
      - `yield` from queue if non-empty, otherwise `yield await new Promise` that stores resolver
    - Call `query()` from Agent SDK:
      ```typescript
      const q = query({
        prompt: messageStream(),
        options: {
          cwd,
          systemPrompt: {

          },
          allowedTools: ['Read', 'Write', 'Edit', 'Glob', 'Grep', 'Bash'],
          settingSources: ['project'],
        },
      });
      ```
    - Return `{ sendMessage, close: () => q.close() }` where sendMessage pushes to queue or resolves pending promise

    **src/cli/utils/process.ts:**

    Import `kill` from `tree-kill` (default import).

    Export interface `ManagedProcesses`:
    ```typescript
    export interface ManagedProcesses {
      devServer?: { pid: number };
      claudeSession?: { close: () => void };
      electronProcess?: { pid: number };
    }
    ```

    Module-level `let shuttingDown = false;`

    Export `function registerShutdownHandlers(processes: ManagedProcesses): void`:
    - Define inner `async function shutdown(signal: string)`:
      - If `shuttingDown` is true, return immediately (idempotent guard)
      - Set `shuttingDown = true`
      - Log `\n  Shutting down ({signal})...` using picocolors.dim
      - Step 1: Close Claude session via `processes.claudeSession?.close()` wrapped in try/catch (ignore errors)
      - Step 2: Kill Electron process tree via `kill(processes.electronProcess.pid, 'SIGTERM')` if pid exists
      - Step 3: Kill dev server process tree via `kill(processes.devServer.pid, 'SIGTERM')` if pid exists (per PROC-02: tree-kill, not process.kill)
      - Step 4: `setTimeout(() => process.exit(1), 5000).unref()` force exit after 5s
      - Step 5: `process.exit(0)` for normal shutdown
    - Register `process.on('SIGINT', () => shutdown('SIGINT'))` (per PROC-03)
    - Register `process.on('SIGTERM', () => shutdown('SIGTERM'))` (per PROC-03)
    - Register `process.on('uncaughtException', (err) => { console.error('Uncaught exception:', err); shutdown('uncaughtException'); })`

    Export `function resetShutdownState(): void` (for testing only):
    - Set `shuttingDown = false`
    - Remove SIGINT and SIGTERM listeners that were added

# Phase 4 -- UI Design Contract

> Visual and interaction contract for the task sidebar, status feedback, error/retry UX, and Claude Code integration pipeline. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (vanilla HTML/CSS/TS in Electron WebContentsView) |
| Icon library | Inline SVG (hand-drawn, matching Phase 2/3 overlay icons) |
| Font | System font stack: `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Helvetica, Arial, sans-serif` |

**Rationale:** Phase 4 continues the vanilla overlay pattern established in Phases 2 and 3. The sidebar is rendered in its own WebContentsView (separate from the overlay) to persist across overlay bounds toggle. Same CSS conventions: dark translucent chrome, white text, inline SVGs, no external dependencies.

## Sidebar Architecture

### Rendering Approach

The sidebar is rendered as a **third WebContentsView** in the BaseWindow, positioned on the right edge. This approach satisfies D-02: the sidebar must persist when the overlay bounds toggle shrinks the overlay to 48x48. Since the sidebar is a separate WebContentsView, its bounds are independent of the overlay view's bounds.

| Property | Value |
|----------|-------|
| WebContentsView | `sidebarView` -- new, added to BaseWindow.contentView |
| HTML entry | `src/renderer/sidebar.html` (new file, electron-vite renderer entry) |
| CSS entry | `src/renderer/sidebar.css` (new file) |
| JS entry | `src/renderer/sidebar.ts` (new file) |
| Preload | `src/preload/sidebar.ts` (new file, contextBridge for sidebar IPC) |
| Background | `#00000000` (transparent, matching overlay view pattern) |
| Z-order | Added after overlay view (`contentView.addChildView(sidebarView)` last) |

### Bounds Management

| State | Sidebar View Bounds |
|-------|---------------------|
| Hidden (no tasks yet) | `{ x: 0, y: 0, width: 0, height: 0 }` -- zero-size, invisible |
| Minimized (icon + badge) | `{ x: windowWidth - 52, y: (windowHeight / 2) - 18, width: 52, height: 36 }` |
| Expanded | `{ x: windowWidth - 296, y: 0, width: 296, height: windowHeight }` |

The sidebar view bounds are managed from the main process (like the overlay bounds toggle). The sidebar renderer sends `sidebar:expand` / `sidebar:collapse` IPC messages; the main process calls `sidebarView.setBounds()`.

The `syncBounds()` function in `window.ts` is extended to also reposition the sidebar view on window resize.

## Interaction States

### State: Hidden (default, before first submit)

- Sidebar WebContentsView has zero-size bounds (effectively does not exist)
- No icon, no badge, nothing visible
- D-03: sidebar does not exist until first instruction is submitted

### State: Minimized (icon + badge, after first submit)

**Visual:**

```
        +------+
        | [>]  |    <- vertically centered on right edge
        | 3/5  |    <- badge counter (only if tasks > 0)
        +------+
```

| Property | Value |
|----------|-------|
| Position | Right edge of window, vertically centered (D-05) |
| Icon container | 36x36px, `rgba(10, 10, 10, 0.88)` background, `border-radius: 12px` |
| Icon | Inline SVG chevron pointing left (14x14px), `rgba(255, 255, 255, 0.55)` |
| Icon hover | Icon color `#ffffff`, background `rgba(255, 255, 255, 0.15)` |
| Badge | Below icon, `rgba(10, 10, 10, 0.88)` background, `border-radius: 8px`, padding `2px 8px` |
| Badge text | "3/5" format (completed/total), 12px semibold, `rgba(138, 180, 248, 1)` |

**Behavior:**
- Click icon to expand sidebar (D-04)
- Badge pulses on task completion or error while minimized (D-07)
- D-18: errors pulse with error color but do NOT auto-expand

### State: Expanded (full sidebar panel)

**Visual:**

```
+--site content (narrowed)--+--sidebar (280px)----+
|                           |  Tasks         [-]  |
|                           |                     |
|                           |  "Make header red"  |
|                           |  [Editing]          |
|                           |                     |
|                           |  "Fix footer link"  |
|                           |  [Done]     [x]     |
|                           |                     |
|                           |  "Align logo"       |
|                           |  [Error] msg        |
|                           |  [Retry] [Dismiss]  |
|                           |                     |
+---------------------------+---------------------+
```

| Property | Value |
|----------|-------|
| Width | 280px (plus 16px total horizontal padding inside = 264px content width) |
| Height | Full window height |
| Background | `rgba(10, 10, 10, 0.88)` |
| Border-left | `1px solid rgba(255, 255, 255, 0.08)` |
| Border-radius | `0` (flush with window edge, no rounded corners on full-height panel) |
| Header | "Tasks" label (11px, medium, `rgba(255, 255, 255, 0.5)`) + minimize button (top-right) |
| Header padding | `16px 12px 8px 12px` |
| Minimize button | 28x28px, `rgba(255, 255, 255, 0.4)` icon, hover `rgba(255, 255, 255, 0.7)` |
| Task list | Scrollable if overflow, `overflow-y: auto`, scrollbar hidden (overlay context) |
| Shadow | `0 0 24px rgba(0, 0, 0, 0.4)` on the left edge (depth cue over site content) |

**Site view adjustment:** When sidebar expands, the site WebContentsView width shrinks by 280px. When sidebar collapses, site view returns to full width. This prevents the sidebar from obscuring site content during active editing review. Overlay view bounds also adjust to match the narrowed site width (if overlay is active).

### State: Auto-Expand (D-06)

When a new task is submitted:
1. Sidebar expands to show the new task row
2. After 2000ms, sidebar auto-minimizes back to badge
3. If user interacts with sidebar during auto-expand (hover or click), cancel the auto-minimize timer

### State: Selection-Mode Minimize (D-08)

When the user enters selection mode (overlay activates):
1. Sidebar auto-minimizes to icon + badge
2. After submit, sidebar re-expands to show the new task

## Animation & Transition Contract

| Element | Property | Duration | Easing |
|---------|----------|----------|--------|
| Sidebar expand | transform (translateX 100% -> 0) | 200ms | ease-out |
| Sidebar collapse | transform (translateX 0 -> 100%) | 150ms | ease-in |
| Site view width adjust | width | 200ms | ease-out (synced with sidebar expand) |
| Minimized badge pulse (completion) | border-color opacity | 600ms | ease-out (single pulse, no loop) |
| Minimized badge pulse (error) | border-color opacity | 600ms | ease-out (single pulse, no loop) |
| Status badge "Sending" pulse | opacity (0.6 -> 1 -> 0.6) | 1500ms | ease-in-out, looping while active |
| Status badge "Editing" pulse | opacity (0.6 -> 1 -> 0.6) | 1500ms | ease-in-out, looping while active |
| Task row enter | opacity (0 -> 1), translateY (8px -> 0) | 150ms | ease-out |
| Dismiss button appear (row hover) | opacity (0 -> 1) | 80ms | ease-out |
| Auto-expand delay | -- | 2000ms hold before auto-minimize | -- |

**No spring animations. No bouncing.** The sidebar must feel utility-first: fast to open, fast to close, out of the way. Transitions exist to provide spatial context (sidebar slides from right), not for delight.

**`prefers-reduced-motion: reduce`:** Disable all transitions. Sidebar appears/disappears instantly. Badge pulse replaced with a static border color change that persists for 2 seconds.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | Tab into sidebar when expanded. Tab order: minimize button, then task rows (retry/dismiss buttons within each). |
| Screen reader labels | Expand icon: `aria-label="Show task sidebar"`. Minimize button: `aria-label="Minimize task sidebar"`. Each task row: `aria-label="{instruction text}, status: {status}"`. |
| Focus management | When sidebar expands, focus moves to minimize button. When sidebar collapses, focus moves to expand icon. |
| Focus trap | No focus trap -- sidebar is a panel, not a modal. User can tab out to the overlay/toolbar. |
| Live region | Task list container: `aria-live="polite"`. New tasks and status changes announced to screen readers. |
| Color contrast | White text on dark chrome: contrast ratio >= 7:1 (WCAG AAA). Accent/error on dark chrome: contrast ratio >= 4.5:1 (WCAG AA). |
| Motion sensitivity | All transitions under 200ms (except badge pulse at 600ms single-shot). `prefers-reduced-motion: reduce` disables all. |

## Z-Index Contract

The sidebar is a separate WebContentsView, so z-index is managed at the Electron level (view stacking order) rather than CSS. Within the sidebar renderer:

| Element | z-index |
|---------|---------|
| Sidebar panel background | 0 (base) |
| Task rows | 1 |
| Sidebar header (sticky) | 10 |
| Minimize button | 10 |

The sidebar WebContentsView is added after the overlay view, making it the topmost view in the BaseWindow.

## IPC Contract (New Channels)

| Channel | Direction | Payload | Purpose |
|---------|-----------|---------|---------|
| `sidebar:expand` | Sidebar renderer -> Main | `void` | Expand sidebar, adjust view bounds |
| `sidebar:collapse` | Sidebar renderer -> Main | `void` | Collapse sidebar, restore view bounds |
| `sidebar:task-update` | Main -> Sidebar renderer | `{ id, instruction, status, error? }` | Add or update a task row |
| `sidebar:task-dismiss` | Sidebar renderer -> Main | `{ id }` | Remove task from list |
| `sidebar:task-retry` | Sidebar renderer -> Main | `{ id }` | Retry errored task |
| `overlay:prefill-instruction` | Main -> Overlay renderer | `{ instruction }` | Prefill input bar for retry (D-19) |
| `claude:submit` | Main (internal) | `{ taskId, instruction, screenshot, dom, bounds }` | Route instruction to Claude agent |

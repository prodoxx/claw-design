# Phase 5 -- UI Design Contract

> Visual and interaction contract for viewport switching, error notifications, splash screen, tooltips, and visual consistency polish. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (vanilla HTML/CSS/TS in Electron WebContentsView) |
| Icon library | Inline SVG (hand-drawn, matching Phase 2/3/4 overlay icons) |
| Font | System font stack: `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Helvetica, Arial, sans-serif` |

**Rationale:** Phase 5 continues the vanilla overlay pattern established in Phases 2-4. All new UI elements (viewport buttons, toast notifications, splash screen, tooltips) use the same dark chrome aesthetic, inline SVGs, and system font stack. No external UI dependencies.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tooltip arrow offset, toast icon-to-text gap |
| sm | 8px | Toolbar divider vertical margin, toast internal padding (vertical), tooltip padding |
| md | 16px | Toast horizontal padding, splash content padding, banner padding |
| lg | 24px | Splash logo-to-text gap, banner action button margin |
| xl | 32px | Splash vertical centering offset |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Toolbar viewport buttons: 36x36px (matches existing toolbar button size)
- Toolbar divider: 1px height, 24px width (visual separator between selection and viewport groups)
- Toast notification max width: 400px
- Splash screen spinner: 24x24px
- Viewport dark surround fill: full window minus constrained site view

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Toast notification message text, splash loading text |
| Label | 11px | 400 (regular) | 1.36 | Tooltip text, pre-flight error hint text |
| Heading | 16px | 600 (semibold) | 1.25 | Splash "claw-design" brand text, persistent error banner title |
| Small | 12px | 400 (regular) | 1.5 | Splash URL text, toast action button text, banner message body |

**Weight rationale:** Two weights only (consistent with Phase 3/4). 400 (regular) for all body text, labels, tooltips, and descriptions. 600 (semibold) reserved for the splash brand text and error banner titles -- elements that anchor visual hierarchy.

**Font rendering:** `-webkit-font-smoothing: antialiased; -moz-osx-font-smoothing: grayscale` on all text. Monospace for splash URL: `'SF Mono', 'Fira Code', 'Cascadia Code', monospace`.

## Viewport Switching

### Toolbar Extension (D-01)

The existing vertical pill toolbar is extended with 3 viewport preset buttons, separated from the selection controls by a visual divider.

```
+------+
| .... |   <- drag handle
| [R]  |   <- region select
| [E]  |   <- element select
|------|   <- divider (1px line, 24px wide, rgba(255,255,255,0.1))
| [D]  |   <- desktop viewport (active by default)
| [T]  |   <- tablet viewport
| [M]  |   <- mobile viewport
+------+
```

| Property | Value |
|----------|-------|
| Divider | `1px solid rgba(255, 255, 255, 0.1)`, width `24px`, centered horizontally |
| Divider margin | `4px 0` (top and bottom, matching toolbar gap) |
| Button size | 36x36px (matches existing toolbar buttons) |
| Button background | `transparent` (matches existing) |
| Button border-radius | `12px` (matches existing) |
| Inactive icon color | `rgba(255, 255, 255, 0.55)` (matches existing) |
| Hover icon color | `#ffffff` with background `rgba(255, 255, 255, 0.15)` (matches existing) |
| Active icon color | `rgba(138, 180, 248, 1)` with background `rgba(138, 180, 248, 0.15)` (reuses `claw-toolbar-btn--active` class) |

### Viewport Presets (D-03)

| Preset | Width | Height | Icon Description |
|--------|-------|--------|------------------|
| Desktop | 1280px | 800px | Monitor outline (20x20 viewBox) |
| Tablet | 768px | 1024px | Tablet outline, portrait orientation (20x20 viewBox) |
| Mobile | 375px | 812px | Phone outline, portrait orientation (20x20 viewBox) |

### Viewport Constraint Behavior (D-02, D-06, D-07)

When a non-desktop viewport is active:

1. **siteView** is resized via `siteView.setBounds()` to the preset dimensions, centered within the window content area.
2. **Window stays full size** -- only the siteView shrinks.
3. **Dark surround** (`#1a1a1a`) fills the area outside the constrained siteView. Implemented by setting the BaseWindow `contentView` background color to `#1a1a1a`.
4. **Overlay covers full window** (D-07) -- selection, toolbar, sidebar all continue to work over the dark surround area.
5. When window is smaller than the viewport preset, siteView fills the window (no constraint applied).

### Viewport Resize Animation (D-05)

| Property | Duration | Easing |
|----------|----------|--------|
| siteView bounds change | 250ms | ease-in-out |

Implementation: Animate via stepped `setBounds()` calls using `requestAnimationFrame` over 250ms, interpolating x/y/width/height from current to target bounds. Not CSS -- this is Electron-level bounds animation.

### Viewport IPC

| Channel | Direction | Payload | Purpose |
|---------|-----------|---------|---------|
| `viewport:set` | Overlay renderer -> Main | `{ preset: 'desktop' \| 'tablet' \| 'mobile' }` | Switch viewport preset |
| `viewport:changed` | Main -> Overlay renderer | `{ preset: string }` | Confirm active viewport (for icon state sync) |

## Toast Notification System (D-08, D-09, D-10)

### Architecture

Toast notifications render in the **overlay WebContentsView** (not a new view). They appear as absolutely positioned elements within the overlay renderer. This keeps them visible regardless of viewport constraint state.

### Visual

```
+---------------------------------------------------+
|  [!]  Dev server crashed. Restart clawdesign.      |
|       [Dismiss]                                    |
+---------------------------------------------------+
```

| Property | Value |
|----------|-------|
| Position | Top-center of window, 16px from top edge |
| Max width | 400px |
| Background | `rgba(10, 10, 10, 0.92)` (slightly more opaque than toolbar for prominence) |
| Border | `1px solid rgba(255, 255, 255, 0.08)` |
| Border-left | `2px solid {severity color}` (replaces generic left border) |
| Border-radius | `12px` |
| Padding | `12px 16px` |
| Box shadow | `0 4px 24px rgba(0, 0, 0, 0.4)` |
| Layout | Flex row: icon (20x20) + gap 8px + content column |
| Icon | Inline SVG, 20x20px, colored by severity |
| Message font | 14px, regular (400), `rgba(255, 255, 255, 0.85)`, line-height 1.5 |
| z-index | 300 (above input bar at 200, above selection at 100) |

### Toast Types

| Type | Duration | Dismissal | Icon | Severity Color |
|------|----------|-----------|------|---------------|
| Non-critical (auto-dismiss) | 5000ms | Auto-dismiss + click dismiss | Info/warning circle | Info blue or warning amber |
| Critical (persistent banner) | No auto-dismiss | Manual dismiss button only | Error triangle | Error red |

### Non-Critical Toast (D-09)

- Auto-dismisses after 5000ms
- Enter animation: `opacity 0->1, translateY -8px->0`, 200ms ease-out
- Exit animation: `opacity 1->0, translateY 0->-8px`, 150ms ease-in
- Clicking anywhere on the toast dismisses it immediately
- Multiple toasts stack vertically with 8px gap (newest on top)

### Persistent Error Banner (D-09, D-11)

For critical errors (dev server crash), the notification stays until resolved or dismissed.

```
+-------------------------------------------------------+
|  [!]  Dev server disconnected                          |
|       The dev server process exited unexpectedly.      |
|       Restart clawdesign to continue.     [Dismiss]    |
+-------------------------------------------------------+
```

| Property | Value |
|----------|-------|
| Same visual as toast | But no auto-dismiss timer |
| Title | 14px, semibold (600), `rgba(255, 255, 255, 0.9)` |
| Body | 12px, regular (400), `rgba(255, 255, 255, 0.7)` |
| Dismiss button | Text button, "Dismiss", 12px regular, `rgba(255, 255, 255, 0.4)`, hover `rgba(255, 255, 255, 0.7)` |
| Position | Same top-center position as toasts, but persists |

### Toast IPC

| Channel | Direction | Payload | Purpose |
|---------|-----------|---------|---------|
| `toast:show` | Main -> Overlay renderer | `{ id, severity, title?, message, persistent, action? }` | Show a notification |
| `toast:dismiss` | Main -> Overlay renderer | `{ id }` | Dismiss a specific notification |

## Splash Screen (D-21)

### Architecture

The splash screen is rendered by the **siteView** itself before the user's localhost URL loads. When the Electron window opens, the siteView first loads a local splash HTML file. Once the dev server URL responds, the siteView navigates to the actual localhost URL. The splash transitions out as the site loads in.

Alternative: Load the splash as a data URL or inject it via `loadURL('data:text/html,...')` to avoid a file dependency. Either approach works; the key behavior is that siteView shows something meaningful during the loading gap.

### Visual

```
+-------------------------------------------+
|                                           |
|                                           |
|           claw-design                     |
|           [spinning indicator]            |
|           Loading localhost:3000...        |
|                                           |
|                                           |
+-------------------------------------------+
```

| Property | Value |
|----------|-------|
| Background | `#1a1a1a` (matches viewport surround color) |
| Brand text | "claw-design", 16px semibold, `rgba(255, 255, 255, 0.9)`, centered |
| Spinner | CSS-only rotating ring, 24x24px, 2px stroke, `rgba(138, 180, 248, 0.8)` |
| Spinner animation | `rotate 360deg`, 800ms, linear, infinite |
| Loading text | "Loading localhost:{port}...", 12px regular, `rgba(255, 255, 255, 0.4)`, monospace font |
| Layout | Flex column, centered vertically and horizontally, gap 16px between elements |
| Transition out | `opacity 1->0`, 200ms ease-out, then navigate to site URL |

### Splash Duration

- Shows until the dev server URL successfully loads in the siteView
- If the site takes >2s to load, the splash provides visual assurance that something is happening
- No minimum display time -- if the site loads instantly, splash transitions out immediately

## Tooltips (D-22)

### Architecture

Tooltips render in the overlay WebContentsView as absolutely positioned elements, triggered by `mouseenter` on toolbar buttons.

### Visual

```
    +-------------------+
    | Region select     |
    +-------------------+
              |
            [btn]
```

| Property | Value |
|----------|-------|
| Background | `rgba(10, 10, 10, 0.95)` |
| Border-radius | `8px` |
| Padding | `6px 10px` |
| Font | 11px, regular (400), `rgba(255, 255, 255, 0.9)`, line-height 1.36 |
| Box shadow | `0 2px 8px rgba(0, 0, 0, 0.3)` |
| Position | To the left of the toolbar button, vertically centered, 8px gap |
| z-index | 10000 (above all overlay content) |
| Max width | 160px |

### Behavior

- Appear after 400ms hover delay (not instant -- prevents flicker during fast mouse movement)
- Disappear immediately on `mouseleave` (no exit delay)
- No animation on enter or exit -- instant appear/disappear (per D-22: "no fancy animations")
- Hidden while any selection mode is active (tooltips would interfere with selection drawing)
- Hidden while overlay is inactive (toolbar is small, tooltips not useful)

### Tooltip Labels

| Button | Tooltip Text |
|--------|-------------|
| Region select | "Region select" |
| Element select | "Element select" |
| Desktop viewport | "Desktop (1280 x 800)" |
| Tablet viewport | "Tablet (768 x 1024)" |
| Mobile viewport | "Mobile (375 x 812)" |

## Error UX Polish (D-08 through D-11)

### CLI Error Messages

All CLI errors use the existing `printError(title, message, suggestion)` pattern from `src/cli/utils/output.ts`. Phase 5 adds pre-flight checks (D-14) and polishes existing error messages.

### Pre-Flight Checks (D-14)

Run at the start of `clawdesign start`, before any spinner output:

| Check | Error Title | Error Message | Suggestion |
|-------|-------------|---------------|------------|
| Node >= 20 | "Node.js 20+ required" | "Found Node {version}." | "Update Node.js: https://nodejs.org" |
| Electron binary exists | "Electron not found" | "The Electron binary is missing from the installation." | "Reinstall: npm install -g claw-design" |
| Claude Code in PATH | "Claude Code not found" | "Claude Code CLI is not installed or not in PATH." | "Install: https://claude.ai/download" |

Note: The Claude Code check already exists in `start.ts`. Phase 5 adds the Node version check and Electron binary check before it.

### In-Window Error Scenarios

| Scenario | Toast Type | Title | Message | Action |
|----------|-----------|-------|---------|--------|
| Dev server crashed | Persistent | "Dev server disconnected" | "The dev server process exited unexpectedly. Restart clawdesign to continue." | Dismiss |
| Dev server connection lost | Persistent | "Connection lost" | "Cannot reach localhost:{port}. Check if the dev server is still running." | Dismiss |
| Claude Code error (already handled by Phase 4 sidebar) | -- | -- | -- | -- |

### Error Message Style

Structured format. Title is a short noun phrase describing the problem. Message explains what happened. Suggestion (CLI) or action (in-window) tells the user what to do next. No exclamation marks. No "Oops" or "Uh oh". Direct and informative.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Viewport buttons | Each has `aria-label` ("Desktop viewport", "Tablet viewport", "Mobile viewport") and `title` attribute for tooltip |
